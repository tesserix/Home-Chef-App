Notes on scope:
- Templates: All emails are inline Go strings in `apps/api/services/email_templates.go` and `apps/api/services/email.go` — no template files (`.tmpl`/`.html`) exist on disk. Brand is mixed: HTML chrome says "Fe3dr" (header, footer, domain `fe3dr.com`), while NATS-published in-app + push copy still says "HomeChef". This brand-string drift is flagged where present.
- Push notifications: Two layers exist — (a) NATS handlers in `services/notifications.go` create both a DB `Notification` row AND publish a `push`-typed `NotificationEvent`; (b) a parallel `push-workers` queue in `handlers/notifications.go::RegisterPushConsumers` publishes its own direct FCM push for `chef_new_order`, `delivery_assigned`, `order_updated`. Both layers fire — so some surfaces have duplicate user-visible text (e.g. "New Order Received" via notif-workers AND "New Order" via push-workers). Both are inventoried separately.
- SMS: Twilio integration exists in `services/notifications.go::sendSMSNotification`, but it has no dedicated template — it just concatenates `"<Title>: <Message>"` from whatever notification event is pulled. So no distinct SMS copy strings exist; all SMS text derives from push titles/messages already inventoried.
- Error strings: 363 unique `gin.H{"error": "..."}` strings exist across handlers. The inventory below includes the user-facing ones; generic "Failed to fetch/create/update X" wrappers (over 100 variants of the same template) are grouped under a single representative row per pattern to avoid noise. The full raw list is preserved in the source files.
- i18n: No i18n / locale / translation system in `apps/api/`. `go-playground/locales` appears only as a transitive dep of the validator. All copy is hardcoded English.
- Legal: Email footer links to `fe3dr.com/privacy` and `fe3dr.com/terms`. NO order-confirmation email contains GST/VAT/HSN tax-line breakdown — the order confirmation lists only items + total (no tax breakdown, no chef name, no delivery address, no refund/cancellation policy). Refund-related copy is absent from emails entirely. Food-safety / allergen language is absent from all emails. These gaps are flagged in the notes column.
