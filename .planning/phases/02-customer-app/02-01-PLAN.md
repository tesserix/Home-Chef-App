## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Onboarding form → API | User-supplied firstName, lastName, phone, address, preferences cross to Go API |
| auth-store → SecureStore | onboardingComplete flag persisted on device |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | Onboarding form (user-info.tsx) | mitigate | Zod schema validates phone (Indian regex), name min 2 chars, pincode 6 digits at field level before API call |
| T-02-01-02 | Spoofing | onboardingComplete flag in SecureStore | accept | Flag is non-sensitive (worst case: user skips onboarding wizard). Backend validates profile completeness on order creation |
| T-02-01-03 | Information Disclosure | address fields in router params | mitigate | Route params are transient (in-memory), not logged, not persisted until POST /v1/customer/onboarding succeeds |
| T-02-01-04 | Elevation of Privilege | Onboarding gate in _layout.tsx | mitigate | Gate checks both isAuthenticated AND !onboardingComplete; neither alone grants access to tabs |
</threat_model>
