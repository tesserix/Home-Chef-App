CRITICAL: Fleet endpoints return 403 for non-staff drivers. Must handle gracefully — do NOT crash.

```typescript
const { data: fleet, error, isLoading } = useQuery({
  queryKey: ['driver', 'fleet', 'overview'],
  queryFn: async () => {
    try {
      const r = await api.get('/delivery/staff/fleet/overview');
      return r.data;
    } catch (e: any) {
      if (e?.response?.status === 403) return null;  // not-authorized, not an error
      throw e;
    }
  },
});
const { data: partners } = useQuery({
  queryKey: ['driver', 'fleet', 'partners'],
  queryFn: async () => {
    try {
      const r = await api.get('/delivery/staff/fleet/partners');
      return r.data;
    } catch (e: any) {
      if (e?.response?.status === 403) return null;
      throw e;
    }
  },
  enabled: fleet !== null, // only fetch if overview succeeded
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fleet/staff API → display | Sensitive fleet data (driver counts, earnings) visible only to staff role |
| profile update → API | Driver-controlled fields (name, city, phone) cross to API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06-01 | Elevation of Privilege | Fleet/staff endpoints | mitigate | API enforces SPViewFleet / SPViewStaff permissions via middleware; client 403 handling shows lock screen — does NOT try to bypass or retry with elevated claims |
| T-03-06-02 | Information Disclosure | Fleet overview shows driver count + earnings | accept | Only visible to users with SPViewFleet permission (API-enforced); non-fleet-managers see lock screen |
| T-03-06-03 | Tampering | PUT /delivery/profile name/city/phone | mitigate | Fields validated with Zod on client; server validates on receipt; min/max length enforced |
| T-03-06-04 | Information Disclosure | Earnings history (weekly amounts) | accept | User's own earnings — legitimately private to the driver; API uses RequireDelivery to ensure only the driver sees their own data |
| T-03-06-05 | Spoofing | Staff invite (POST /delivery/staff/invitations) | mitigate | API requires SPInviteStaff permission; client 403 response handled gracefully — non-manager sees lock screen for entire staff page |
</threat_model>
