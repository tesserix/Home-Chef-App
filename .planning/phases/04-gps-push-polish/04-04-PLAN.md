**Step 1 — Create SkeletonCard component (apps/mobile-customer/components/SkeletonCard.tsx):**

```typescript
import React from 'react';
import { View } from 'react-native';

interface SkeletonCardProps {
  count?: number;       // How many skeleton rows to render (default: 4)
  height?: number;      // Height of each card (default: 96 = h-24)
  rounded?: boolean;    // Whether to use rounded-full for circular skeletons
}

export function SkeletonCard({ count = 4, height = 96, rounded = false }: SkeletonCardProps) {
  return (
    <>
      {Array.from({ length: count }).map((_, i) => (
        <View
          key={i}
          style={{ height }}
          className={`bg-gray-200 w-full mb-3 animate-pulse ${rounded ? 'rounded-full' : 'rounded-xl'}`}
        />
      ))}
    </>
  );
}
```

**Step 2 — Create OfflineBanner component (packages/mobile-shared/src/components/OfflineBanner.tsx):**

```typescript
import React, { useEffect, useState } from 'react';
import { View, Text } from 'react-native';
import NetInfo from '@react-native-community/netinfo';

export function OfflineBanner() {
  const [isOffline, setIsOffline] = useState(false);

  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener((state) => {
      setIsOffline(!state.isConnected);
    });
    return unsubscribe;
  }, []);

  if (!isOffline) return null;

  return (
    <View className="bg-orange-500 px-4 py-2 flex-row items-center justify-center">
      <Text className="text-white text-sm font-medium">
        You are offline — showing cached data
      </Text>
    </View>
  );
}
```

**Haptic feedback rules per D-20:**
- Order placed (checkout success) → `Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success)` — strongest success signal
- Accept order / reject order (vendor) → `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium)` — decisive but not alarming
- Delivery status transition (picked up, in transit, delivered) → `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium)`
- Delivery complete (delivered) → `Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success)` — upgraded to success notification style
- Payment success (same as order placed) → `Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success)`

All haptic calls MUST be:
1. Fire-and-forget: `void Haptics.impactAsync(...)` or `Haptics.impactAsync(...).catch(() => {})` — never `await` in a hot path
2. Added to the SUCCESS callback of the mutation/action, not the onPress handler (avoid haptic on error)
3. Wrapped in a try-catch to avoid crashing if haptics are unavailable (simulator)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NetInfo → UI state | Network state from OS; no external trust boundary; purely presentational |
| React Query cache | Cached data shown offline; stale data displayed is low risk for read-only views |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-18 | Information Disclosure | Cached React Query data shown offline | accept | Cached order/menu data is already visible to authenticated user; stale display for short offline periods is acceptable; accept |
| T-04-19 | Denial of Service | Pull-to-refresh spamming | accept | React Query's `isRefetching` gate prevents duplicate simultaneous refetches; accept |
| T-04-20 | Spoofing | OfflineBanner UI state | accept | NetInfo event is read-only from OS; no user input; no spoofing vector; accept |
