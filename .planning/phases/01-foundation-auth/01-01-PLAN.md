**metro.config.js (identical for all three apps — only __dirname path changes automatically):**
```javascript
const { getDefaultConfig } = require('expo/metro-config');
const { FileStore } = require('metro-cache');
const path = require('path');

const projectRoot = __dirname;
const monorepoRoot = path.resolve(projectRoot, '../..');

const config = getDefaultConfig(projectRoot);

config.watchFolders = [monorepoRoot];
config.resolver.nodeModulesPaths = [
  path.resolve(projectRoot, 'node_modules'),
  path.resolve(monorepoRoot, 'node_modules'),
];
config.resolver.unstable_enableSymlinks = true;
config.resolver.disableHierarchicalLookup = true;

config.cacheStores = [
  new FileStore({ root: path.join(projectRoot, '.metro-cache') }),
];

module.exports = config;
```
Note: `disableHierarchicalLookup: true` prevents Metro finding duplicate React Native instances across pnpm symlinks — omitting it causes "Invariant Violation: No React Native renderer" crash.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app.json → EAS Build | Bundle identifiers and schemes are locked into binaries at build time |
| metro.config.js → workspace packages | Metro resolves symlinked packages from pnpm; incorrect config resolves wrong React Native |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | metro.config.js | mitigate | Enforce `disableHierarchicalLookup: true` to prevent Metro resolving duplicate React Native from nested node_modules — duplicate RN instance is an invariant violation |
| T-01-02 | Tampering | Deep link schemes | mitigate | Distinct schemes per app (D-14): homechef-customer, homechef-vendor, homechef-delivery — prevents intent hijacking where one app intercepts another's deep links |
| T-01-03 | Information Disclosure | app.json infoPlist | accept | NSFaceIDUsageDescription and NSCameraUsageDescription are public metadata required by App Store — no sensitive data |
| T-01-04 | Elevation of Privilege | EAS build profile | mitigate | developmentClient only in development profile — production profile has no dev tools, no debug menu |
</threat_model>
