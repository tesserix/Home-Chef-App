# Customer App v2 Redesign Plan — Airbnb language

The customer app gets a premium consumer-marketplace look in the Airbnb mold: white, airy,
photo-forward, **rausch coral** as the single accent. Same execution model that worked for the
vendor v2 (spec doc → parallel waves → live verify on sim → commit per wave).

## 1. Palette & tokens (Wave 0 — foundation, do first)

Add a **customer palette** to `packages/mobile-shared/src/theme/tokens.ts` (additive — vendor
and driver untouched):

| Token | Value | Use |
|---|---|---|
| `coral.DEFAULT` | `#FF385C` | Primary CTA, links, selected states, heart-saved, focus |
| `coral.pressed` | `#E00B41` | Pressed/hover |
| `coral.tint` | `#FFE8EC` | Chip/badge tint backgrounds |
| `charcoal` | `#222222` | Primary text + dark buttons (Airbnb's near-black) |
| `gray.secondary` | `#717171` | Secondary text, captions |
| `gray.hairline` | `#EBEBEB` | Dividers, card borders |
| `canvas` | `#FFFFFF` | Page background (Airbnb is white-first, NOT grey-canvas) |
| `surface.soft` | `#F7F7F7` | Input fills, soft sections, image placeholders |
| `success` | `#008A05` | Delivered/confirmation only |

- Typography stays **Geist + Inter** (Airbnb Cereal is licensed; Geist's geometry reads close).
- Radius: 12 images/cards, full for pills/search; hairline borders over shadows except floating
  elements (search pill, sticky CTA bar, map controls) which get `shadow[2]`.
- `.impeccable.md`: add "Customer palette addendum" (per its update-the-file-first rule).
- Guardrail: customer screens must never use `herb` (persimmon); vendor/driver never use `coral`.

## 2. Signature Airbnb patterns to implement

1. **Floating search pill** (home): radius-full white pill, shadow[2], "What are you craving?" —
   anchors the whole brand impression.
2. **Photo-led chef cards**: full-width food photo (radius 12, 4:3), heart toggle top-right
   (white glyph + charcoal scrim; saved = coral fill with a 150ms scale pop), below: chef name
   semibold + "★ 4.9 (212)" inline, cuisine line gray, price/delivery line charcoal.
3. **Category chip row** under search: icon + label chips, underline-free, selected = charcoal
   text + 2px charcoal underline (Airbnb category bar style).
4. **Chef detail**: full-bleed photo header (~40% viewport) with circular white overlay buttons
   (back / share / heart), content sheet scrolls over it; menu items as photo-right rows;
   **sticky bottom CTA bar** — white, top hairline + shadow, "Add to cart · ₹540" coral button.
5. **Sticky CTA bars** on cart/checkout ("Continue", "Place order · ₹780") — coral filled,
   radius 8, 52pt.
6. **Order tracking**: map full-screen, floating white status card bottom (driver, ETA,
   progress dots in coral), circular floating back button.
7. **Status communication**: tint chips like vendor v2 but coral/charcoal-family; success green
   only for delivered.
8. **Profile/favorites**: iOS grouped lists (white cards on white with hairlines — Airbnb uses
   hairline-separated rows, NOT grey canvas) and a 2-column photo grid for favorites.

## 3. Waves

**Wave 0 — Tokens + spec (small, sequential):**
tokens.ts customer palette · CUSTOMER-UI-SPEC.md (binding doc like vendor's) · .impeccable.md addendum.

**Wave A — The money path (4 parallel agents, disjoint files):**
1. `(tabs)/_layout.tsx` + `(tabs)/index.tsx` (home: search pill, category row, chef cards) + chef card component(s)
2. `chef/[id].tsx` (photo header, menu rows, sticky add-to-cart) + `components/chef/*`
3. `checkout.tsx` + `components/cart/*` (line items, price breakdown, sticky place-order, payment method rows)
4. `(tabs)/favorites.tsx` (photo grid + empty state) + heart behavior shared with home cards

**Wave B — Track + manage (3 parallel agents):**
5. `order/[id]/index.tsx` + `order/[id]/track.tsx` + `components/tracking/*` + `components/orders/*` (status timeline, map overlays)
6. `(tabs)/orders.tsx` + `payment/result.tsx` (history rows, success/failure states)
7. `(tabs)/profile.tsx` + `social.tsx` + `catering.tsx` (grouped lists, feed cards)

**Wave C — Entry + motion + polish (sequential, main session):**
8. `(auth)/*` + `(onboarding)/*` light-touch (accent swap, button language; LoginScreen is
   shared with vendor — needs an accent-color prop or customer-local wrapper, NOT a fork)
9. Motion: staggered card entrances, heart scale-pop, sticky-bar slide-in, pressed scale on
   CTAs (reanimated, ease-out-quart, reduced-motion gated — same recipe as vendor)
10. Live QA on iPhone 17 Pro sim (Metro hot-reload), screenshot review of every screen, fix pass.

## 4. Verification gates

- `cd apps/mobile-customer && npx tsc --noEmit` after every wave — only pre-existing
  duplicate-@types/react error classes allowed (count baseline before Wave 0).
- Visual check on sim per wave; grep gate: no `herb` references introduced in customer screens,
  no `coral` outside `apps/mobile-customer`.
- Commit per wave (conventional, single-line), push at end.

## 5. Risks / notes

- **Shared screens**: LoginScreen/ForgotPassword live in `packages/mobile-shared/src/screens` and
  serve all three apps — parameterize accent via prop/context; do not fork.
- **Token name collision**: customer code must not reach for `herb`; add the coral tokens under a
  clearly customer-named export to keep grep-able separation.
- **Photo dependency**: card quality depends on chef photo quality from the API; placeholder
  treatment (surface.soft + utensil glyph) required for missing images.
- **Map screen** (track.tsx) uses react-native-maps — overlay styling only; don't touch map logic.
- Dark mode stays out of scope (queued phase, both apps together).

## Estimate

Wave 0 ~30 min · Wave A ~1 session (4 agents parallel) · Wave B ~1 session (3 agents) ·
Wave C ~half session. Comparable to the vendor v2 effort (15 files, ~2.5k LOC delta there).
